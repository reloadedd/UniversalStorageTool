1. Abstract

Universal Tool Storage is a web application that gives all its users the possibility to store complex files of a large size through another cloud service provider such as: DropBox, Google Drive, OneDrive (others may be added in time). This document describes how Universal Tool Storage works and should be used.

2. Introduction

The current digital age is marked by our reliance on the cloud. Multiple applications are used in a large diversity that defines the ides of cloud storage.

Universal Storage Tool is the application that helps users managing multiple cloud storage services from a single place. The application allows all its users to upload, download and archive files. Even if a file is lage and can not be uploaded due to its size, Universal Storage Tool will offer the possibility to fragment this file in smaller pieces and upload them in different cloud storage such that they will fit. Of course, if the user wants to download it, will be reconstructed to its initial form and delivered to him/her.

2.1. Motivation

A conducted study shows how "more than half of people use at least 3 different storage services". "Almost all people believe that their data habits have changed due to personal Cloud Storage. A majority of them are now using the cloud as their primary data storage. More than 50% of surveyed people use a free or paid limited storage plan.

2.2. Purpose

Universal Storage Tool has, as a first purpose, to offer to its users a simple and clean platform for managing multiple cloud services by linking them to an account. Thus, the users will not need to worry about the details, just focus on what matter the most for them: their data.

The second purpose of this application is efficiency and time saving for our clients, a better and faster way to storage files across multiple cloud storage services.

3. Structure

Universal Storage Tool is structured based on functionalities that are provided to its users. The functionalities provided by the application include manipulating and computing different operations on files that belong to a logged user. In the following section, all those functionalities will be presented.

3.1. Functionalities

Following a natural order in witch they were implemented, Universal Storage Tool offers:

  • User registration and authentication.
  • Linking a cloud service account that does not exist in the current UnST account.
  • Displaying via UI the user account details: user based info and statistics regarding the amount of used space on every cloud service linked to the current account.
  • Displaying via UI all the files from all linked cloud storage services.
  • Uploading files in a classic manner (in a single linked cloud application) or by fragmenting it and uploading it in multiple linked cloud services in case is too large to fit in any linked cloud application.
  • Downloading an existent file that was uploaded in a classic manner if it was uploaded in the same way, or by reconstructing all its fragments from every linked cloud application and putting them together and after that downloading it.
  • Archiving files using a preset software for this purpose.

3.2. UI components

Universal Storage Tool's structure is comprised by a number of HTML pages, each having different visual components, which we will explain in the current section.

3.2.1. Index page

The index page is the main page the user will see. It will provide navigation functionalities and will allow the user to see all the files associated with his account, choose to upload a new file and see how much space he/she has left in total.

3.2.1.1 Navigation Bar

The Navigation Bar is the UI element providing navigation functionality to the user. It includes an image with the project's logo, which if clicked redirects to the Index page, and more links to the right, to all the other HTML pages of the website.

The responsiveness is provided through the flexbox containers and media queries for small displays. If the display is narrower than 678 pixels, the elements in the right will not be visible anymore. In their place, a menu icon will appear, and by clicking it the links to the other pages will be displayed in a drop menu fashion.

This element appears on all pages, for a more standardized navigation system, except for the Report page, since that one must comply with the Scholarly HTML format.

3.2.1.2 Side Menu

The side menu is one of the two main elements of the body, making one of the two grid columns the body is made of. It allows access all the time to the button for adding a new file and the statistics for the available space in total.

When the display is narrower than 678 pixels, its width shrinks to 42 pixels, and instead of the elements it contained, only a menu button will be visible. Upon clicking the button, the side menu will revert to its original width, allowing the user to select one of the possible actions.

3.2.1.3 Main View

The main view is the other grid element of the body. This is where files and folders will be visible, allowing the user to download them or confirm that his uploads did succeed.

The elements inside this container will be loaded through JavaScript, through AJAX calls using the fetch API, and will be displayed using a simple float left. This way, regardless of the display width, all files will be visible, since it is scrollable.

3.2.2. Login/Register Page

This page provides the functionality for user login and registration. Both the "Login" and "Register" functionalities will be on the same page. The presentation of the page consist of a form with two text box inputs: username and password and two submit buttons: login and register. If a submit button was clicked, a validation of the input data, username and password, will be executed firstly: username needs to have minimum 6 and maximum 16 alphabetic and numeric characters and password needs to have minimum 8 and maximum 32 alphabetic and numeric characters.

Based on what button was clicked, the corresponding action will be done. In case of a "Login" button, the user will request to login as a user with username and password inserted in the input fields of the login form, respectively a user registration will be requested if the pressed button is "Register".

A successful response from the server will redirect the user to the "Home" Page, otherwise the user will be restricted to remain in the "Login/Register" page.

3.2.3. Account Page

The "Account" page offers information regarding the current logged in user: username, the registration date, his/her linked cloud applications and a status bar for each one showing the amount of space already in use.

This page also allows users to edit or delete their account and also add a new link to another cloud storage application.

3.4. Server Side Role - Routes

On the Server Side, Universal Storage Tool has the logic of the application implemented. In this section, we will show you how the server interacts with the client by mostly referring to application routes.

3.4.1. Entry Point of the application, Root Route

On the "/index" or "/" route is the Entry point of UnST application. When the an user enters here, firstly, if he is not logged in, aka does not have its user data saved on browser cookies, It will be redirected on the login page. Otherwise, the needed data from server will be loaded on this page: total available space, all user files and folders and others.

The total available space is provided by the server by requesting all the available space from all the user linked drives and summed up.

All user files name and drivers corresponding ids are stored in the database. So everytime when uploading or deleting a file, that file isn't gone be only uploaded/deleted, but it will also have its name, id, parent folder and driver that is from in the database.

When listing all the files, the server will only list the name of files and directories from database that are in the current directory.

When downloading a selected file, the server will actually request that file with its specific id, from its specific driver/drivers.

Before entering in any route, the server will refresh the access token saved as browser cookie from every linked drive.

3.4.2. Drivers Routes

Every driver that is linked to this app will have access to the following routes "[driver name]/space", "[driver name]/auth", "[driver name]/token", where [driver name] can potentially be "g-drive","onedrive" or "dropbox".

The "/space" from every driver route will give you as response in body the total available space in that specific drive and also the amount of used space from it.

The "/auth" and "/token" from every driver route will be used to grand permission to the user drivers and to get a refresh token that will be stored on the database.

3.4.3. User Routes

The "/login", "/register" and "/logout" are that user specific routes. On login the server will take the username and password and will make a database verification to see if that data is valid and exist in the database. On register the server will add a new user in the database with that specific username and password, only if that user does not already exists in the database.After login or register the user id is saved via browser cookies. On logout the browser cookies are cleared.

3.4.4. File Routes

On "/file" route, depending on the method: delete or patch, the server will delete or rename the specifies file with the file id in the query.

On "/dir" route, depending on the method: delete or patch, the server will delete or rename the specified directory with the directory id in the query.

On "/newDir" route (post) a new directory will be created with the name and parent-directory id specified in the body request.

4. User Manual

In this section we will show you how to use this application as its user.

4.1. Account

4.1.1. Authentication & Registration

Enter on the "/login" path (or "/" or "/index" and will be redirected to "/login" if you are not signed in) and a page form will be shown. If you are register on UnST enter your registration name and password and after that you will be redirected to the home page. If you are not registered, you can do it by completing the form from "/login" and press the "Register" button. Make sure that the username and password fields are completed and not empty and they respect the following properties: the length of the username is at least 6 and maximum 32 characters long and the length of the password is at least 8 and maximum 32 characters long. If the username is already taken, retry another one.

4.1.2. Account Settings

This application also gives you the options to edit your account username and password. And if you want to delete your account, all the files that you stored in this app will be deleted or not from all the drivers the are stored into, according to your decision.

4.2. Files & Directories management

4.2.1 See your listed directories/files

Enter on "/" or "/index" page and in the files directories container will be listed all the files and directories. From here you can navigate through directories and see their content by simply clicking them to open.

4.2.2 How to create a directory

Enter on "/" or "/index" page and in the files directories container will be listed all the files and directories. From here you can create a directory on the current directory that you are into by clicking the "New folder" button situated on the of the left side menu from index page.

4.2.3 How to upload a file

Enter on "/" or "/index" page and click the "Upload Files" button situated in the top part of the side menu.To upload a file or more, select them from the window that popped out after you clicked the "Upload Files" button. The server will handle the request after and give you a response. The uploaded file will be situated on the current directory that you are in.

4.2.4 How to download a file

Enter on "/" or "/index" page and in the files & directories container will be listed all the files and directories. From here you can download a file by right clicking on a file and from the context menu select "Download" and immediately after that the download of that file will be initiated.

4.2.5 How to delete a file/directory

Enter on "/" or "/index" page and in the files & directories container will be listed all the files and directories. From here you can delete a file by right clicking on a file and from the context menu select "Delete" and immediately after that the selected file/directory. If a directory has more files and/or directories and the user deletes it, the operation will be done recursively.

4.3. Drivers Management

4.3.1. Add a drive

To add a drive enter on the "/account" page and click on the "Add Drive" button and select a drive that you want to add from the pool(currently only Google Drive, OneDrive and Dropbox). After that you will be redirected to a page where you authorize us to have access to your drive account that you just logged in. Now, you are able to upload, delete and list files from drivers that you linked to this application.

4.3.2. See Drivers Statistics

If you have drivers from the pool linked to our application, then you can see different statistics regarding the space available on those drives. On "/account" page you have listed the available space on every single drive linked and on the "/index" or "/" page you have the total available space displayed in the bottom of the left side menu.
